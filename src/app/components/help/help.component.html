How to Use

The checklist tool is made up of three main parts:

  • Content selection
  • Compliance level selection
  • The guidelines to test

At any point you can move backwards to any particular stage of the tool by selecting that step from the 3-step navigation at the very top of the tool.

Content Selection

The content selection screen is where you try to best match the content types you wish to check. Here's a better breakdown of what each type covers:

Animation
This covers everything from a full-on animated show piece right, a website with animations as a user scrolls, right the way through to a website using subtle pulsing animation to draw attention to new features.
Audio
Any website that uses audio, including noises attached to events (messages, alerts, etc) and the adio from videos.
Forms
Forms are a crucial part of the web, and this covers all the things you need to check if your content contains them or form components, like buttons, text inputs, checkboxes, and other form-like components.
Images
This covers tradition <img> images as well as <canvas> and <svg>. Even CSS images would be covered under this.
Modals
Modals include things like alerts, prompts, and confirmation dialogs which attempt to trap the user in them until a specific type of action is taken. A regular system modal dialog handles many of the normal accessibility needs of a user, but they often get re-implemented to allow for better customisation.
Timed
If components on your website require a user to be able to take a specific action in a specific timeframe, then this will ensure the generated guidelines cover it. For example, things like drop-down (not <select>) menus which require a mouse movement from point a to b in less than a second would count under this.
Translations
Any content which is available in either multiple languages, or content which includes mixed language content needs some specific language guideline checks.
Video
As well as traditional video content, this also covers things like animations which are video-like.

If your content includes none of these types, then only a few accessibility guidelines which apply to all basic content will be presented.

Compliance level selection

Compliance against the Web Content Accessibility Guidelines (WCAG) ranges from the most basic (A) up to the near impossible (AAA). It's fairly typical to aim for AA compliance, as it's a good middle ground.

AAA compliance is incredibly difficult to achieve, especially if you have live video content. On the other hand, single-A compliance is very easy, but doesn't generate a very big change for those most in need.

The Guidelines to Test

Once you've made your selections, you'll be presented with a selection of guidelines which best match your choices from the WCAG 2.1 specification.

Each guideline is listed under its main groups (perceivable, operable, understandable, & robust), which generally outline what type of thing they check.

Initially, all guidelines are in their minimised state, with only their checkbox, number, and name shown. Each one can be expanded by clicking on the name to show a more detailed description that explains the guideline better. Where possible, there are code snippets which you can copy and paste into your browser console to test the live content of your website. These snippets range from testing images without alternative text, to the colour contrast of text elements on the page.

Saving/Loading

You can save and load a single state using the buttons provided in the toolbar. This functionality has been built with privacy in mind, so state is saved in your browsers local storage rather than in a cookie or directly against a server.

Back to top

Saving

If you have no current saved state, then clicking the save button will do so with only a success notification. However, if you already have a state saved, you will be warned about overwriting this if you continue.

Back to top

Loading

Clicking the load button will always ask for confirmation about loading a state over the current state. This is to ensure that you don't accidentally load a state over the top of any current changes or selections you may have made.
